Einführung in Reinforcement Learning:

Reinforcement-Learning ist eines der interessantesten Felder im Bereich der Knstlichen Intelligenz. In vielen Bereichen zeichnet es sich ab, dass das alleinige Sammeln von Daten nicht ausreichend ist appropriate Ergebnisse und Werte zu erzielen. Das Reinforcement-Learning geht einen anderen Ansatz. Anstatt nur aus Grundlage vieler Daten Muster und Regeln abzuleiten, erschlieen sich diese Muster und Regeln durch eine Art Try and Error Prinzip. Seid den Untersuchungen von Iwan Pawlow kennen wir das Prinzip der Konditionierung. Das "richtige" Verhalten wird belohnt das "falsche" bestraft. Doch wie knnen wir den Bogen zum Reinforcement-Learning im Musikbereich spannen? Daher voher eine kurze Einfhrung.

Das Reinforcement-Learning besteht aus 4 Bestandteilen:

1. Das Environment

Das Environement ist sozusagen die Spielwiese des Modells. Es ist der Rahmen in dem unser "Protagonist" bzw. Agent Aktionen(dazu gleich mehr) ausfhrt. Z.B. kann man sich darunter das Level eines Compuerspiels vorstellen. Wie unten zu sehen eine ganz einfache Darstellung eines Environments im betragenen Sinne.

2. Agent

Der Agent ist die Spielfigur, die sich im Environemt bewegt und Aktionen ausfhrt. Bei Mario Kart wre das z.B. einfach Mario ("wer nimmt Luigi??").

3. Action

Der Agent fhrt Aktionen (Actions) aus. Bei einem einfachen Plattformspiel kann man sich darunter z.B. einfach vorstellen, dass der Agent entweder vorwrts bzw. rckwrts luft, springt oder aber stehenbleibt. Bei Mario Kart wrden hier weitere Aktionen definiert werden...

4. Rewards

Wie oben bereits erwhnt wollen wir nach Pawlow-Manier das Verhalten bzw. die Aktionen des Agents entweder loben oder tadeln. Dies ist mit Rewards mglich. Fr eine richtig ausgefhrte Aktion im Environment erhlt der Agent eine Belohnung (positiven Wert), bei einer falschen Aktion eine Strafe (negativen Wert). Der Agent versucht dabei stets die maximale Anzahl an Rewards zu erhalten bevor er z.B. das Spiel terminiert und merkt sich den Verlauf der Aktionen bei denen er eine Strafe erhalten hat. So wei er es beim nchsten Versuch besser und begeht den selben Fehler nicht nocheinmal.

Reinforcement Learning in der Musik¶

Diese Grundlagen des RL knnen in Bezug zur Musik wunderbar eingesetzt werden. Schon lnger werden Texte und Musik durch Deep-Learning generiert, dabei verwendet man meistens Rekurennte Neuronale Netze und LSTM-Layers. Die Ergebnisse, vor allem bei greren Projekten sind sehr gut, aber die Resultate wirken auch, da sie immer auf alten Daten aufbauen, wie eine Wiederholung von Ideen die schon durchdacht wurden. RL wre hier ein alternativer Ansatz, der sehr vielversprechend ist.

Das Ziel des hier dargestellten Models ist es, die Tonhhen einer Midi-Datei zu extrahieren, die einzelnen Noten zu identifizieren, mit diesen Vorgaben ein Training mit den Tonhhen durchzufhren und dann anschlieend das verwendete Musikstck korrekt widergeben zu knnen.

1. Environment

Unten sehen wir verschiedene Tonhhen, die in einer Midi-Datei codiert sind. Insgesamt gibt es 128 verschiede Tonhhen. Das Environment knnen wir uns als ein Keyboard mit 128 verschiedenen Tasten vorstellen, jede Taste besitzt einen Wert zwischen 0 und 127.

2. Agent

Unser Agent ist jetzt der Klavierspieler, der jede Taste drcken und durch jede Tonhhe springen kann.

3. Actions

Da wir uns auf einem Keyboard bzw. Piano befinden haben wir 3 verschieden Aktionen die der Agent ausfhren kann. Er kann vorwrtsgehen, stehen bleiben oder zurckgehen. Jeder Aktion wird ein Wert zugeordnet:

Bei Forward geht der Agent eine Position nach vorne, bei Stay bleibt der Agent bei seiner Position und bei Backwards geht er eine Position nach hinten.

4. Reward

Nhert sich der Agent der gesuchten Note, so erhlt dieser einen hheren Reward. Entfernt sich der Agent, so verringert sich der Reward. Im Anschluss wird der Output des Modells mit dem Imput verglichen. Stimmen beide berein so erhlt das Modell wiederum einen zustzlichen Reward.
